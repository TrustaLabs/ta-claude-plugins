# 快速开始指南


## 第一次使用

### 1. 触发 Skill

在 Claude Code 中输入以下任何一种表达：

```
帮我制定实现计划
```

或者：

```
我需要创建一个用户认证系统的开发计划
```

### 2. 提供上下文

Skill 会要求你提供更多信息，例如：
- 功能描述
- 相关文档
- 技术约束
- 特殊要求

### 3. 参与对话

Skill 会通过 3-7 轮对话与你互动：

**第1轮：澄清问题**
- 功能边界是否清晰？
- 有没有遗漏的需求？
- 对技术方案有没有偏好？

**第2轮：细化方案**
- 根据你的回答调整计划
- 明确每个阶段的具体任务

**第3轮：讨论权衡**
- 说明方案中的技术取舍
- 讨论备选方案

**第4轮（可选）：补充边缘情况**
- 添加错误处理
- 考虑性能影响
- 补充测试策略

**第5轮（可选）：最终审查**
- 确认计划完整性
- 确保每个阶段可执行

### 4. 获取计划文档

完成对话后，计划文档会保存到：
```
docs/plans/[功能名称]-[时间戳].md
```

## 示例对话

### 示例 1：简单功能

```
用户：帮我制定一个添加"深色模式"功能的计划

Skill：我来帮你制定深色模式功能的实施计划。首先让我了解一些细节：

1. 功能边界：
   - 是否需要支持自动切换（跟随系统）？
   - 是否需要记住用户的选择？
   - 哪些页面需要支持深色模式？

2. 技术偏好：
   - 你们项目使用的是什么 UI 框架？
   - 是否已有主题系统？

用户：
- 需要支持自动切换和手动切换
- 需要记住用户选择
- 所有页面都要支持
- 我们使用 MUI，已有主题系统

Skill：好的，基于你的回答，我来细化实施方案...

[继续对话...]

Skill：计划已完成并保存到 docs/plans/dark-mode-2024-12-24-10-30.md
```

### 示例 2：复杂功能

```
用户：我需要实现一个完整的用户认证系统，包括登录、注册、密码重置

Skill：这是一个复杂的功能，我会通过 5-7 轮对话帮你制定详细计划。

第1轮：让我先了解需求...
[提出澄清问题]

第2轮：基于你的回答，我来细化方案...
[展示详细步骤]

第3轮：我们来讨论技术选型...
[对比 JWT vs Session]

第4轮：让我们补充边缘情况...
[错误处理、安全考虑]

第5轮：最终审查...
[确认完整性]

计划已完成！
```

## 查看计划文档

使用任何文本编辑器打开计划文档：

```bash
# Windows
notepad docs/plans/[文件名].md

# 或使用 VS Code
code docs/plans/[文件名].md
```

## 计划文档包含什么？

1. **需求概述** - 背景、目标、范围
2. **技术方案** - 架构、技术选型
3. **实施步骤** - 分阶段的具体任务
4. **风险评估** - 风险识别和缓解措施
5. **测试策略** - 单元测试、集成测试、E2E 测试
6. **技术权衡** - 方案对比和决策理由
7. **回滚计划** - 如何回滚


## 常见问题

### Q: Skill 没有触发？

A: 确保你的表达包含关键词：
- "制定计划"、"创建计划"
- "分析需求"、"设计方案"
- "评估方案"

### Q: 对话轮数太多/太少？

A: 你可以明确告诉 Skill：
- "这个功能比较简单，3轮就够了"
- "这个功能很复杂，我们需要更详细的讨论"

### Q: 计划文档在哪里？

A: 默认保存在项目根目录的 `docs/plans/` 目录下。

### Q: 可以修改计划吗？

A: 当然可以！计划文档是 Markdown 格式，可以随时编辑。

### Q: 如何与团队分享计划？

A: 将计划文档提交到版本控制：
```bash
git add docs/plans/
git commit -m "Add implementation plan for [feature]"
git push
```
